Data Transmission System 

Implementation of a Data Transmission System for the National Agency for Employment (ANOFM)

  • Project Objective: 

The implementation of a national wide VPN between all the ANOFM Locations. (at signing - 141 locations later extended to 227 locations)

Monitoring the equipment's status/movement and monthly invoicing of the services. (~8 years )

  • Project Description: 

Our company supplied and Installed radio telecom equipment's in each ANOFM location, connected the equipment's to the Mobifon Network and using Mobifon as a subcontractor, we supplied to ANOFM data transmission services and Internet access.

  • Implementation time / Project Duration :

 3 Months (project duration 2004 – 2012)

  • Project revenues :

~2.500.000€

  • Implementation Procedure :

The first step in eecuting this project was the wining of the public bid for it's execution. We saw the public anouncement, we established a strategy for the execution, we analized our resources, we negociated with the suppliers, we drafted the technical offer and financial one. After participating to the tender and winning we establishied the implementation schedule with the Beneficiary’s HQ representatives. (Implementation Stages: site visit, technical design, equipment installation). 

Besides the above steps, my attributions in the project were:

        - coordinatiton of the installation teams (ours and subcontractors);                                                       - sending weekly reports to the beneficiary in order to present the implementation status.           - daily meetings with the head of the installation teams and with the Technical Director                 - negociating the final contractual conditions with the suppliers/subcontractors.                             - monitoring the project’s Cash Flow - invoicing/payments for supplyers/ subcontractors/             beneficiary.

Together with 2 colleagues I created in excel an easy to access data base for tracking all the supplied equipment's. (equipment/serial number/location). This system was later used for tracking the movement of equipment's and of the issued invoices. The invoices were issued per each county.


  • Implementation Challenges/Problems :

In some of the locations there were no IT&C equipment's to be connected to the system and/or the technical solution required a lease contract to be signed with a third party.

We openly communicated these shortcomings with the beneficiary and found solutions for them. (ex: Our installation team made a second trip to the location / We changed for the moment the solution dial-up instead of wireless until all the documents were in order).
